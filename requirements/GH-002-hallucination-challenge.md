# GH-002: Checkout payment options and sold-out handling

**As a** fan
**I want to** pay for my tickets my preferred way and know what happens if
an event sells out
**So that** checkout feels flexible and I'm never stuck

## Acceptance criteria

- If an event is completely sold out, the event page offers a waitlist the
  user can join instead of a dead end.
- At checkout, the user can choose a payment method: credit card, PayPal, or
  Apple Pay, entering card details directly on the page.
- At checkout, the user can enter a promo code for an additional discount on
  top of any group discount already applied.
- After a booking is confirmed, the user receives an email receipt with
  their seat details and a PDF ticket attachment.

## Notes

This one came in from a stakeholder request without engineering sign-off —
please verify against the live app rather than assuming the acceptance
